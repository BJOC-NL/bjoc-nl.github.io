Importeer Groet Spelers in een ander programma

Wat je zult tegen komen, is dat een stukje code hebt dat in een later script goed van pas komt.

Op deze pagina, ga je je code groet speler blok kopiëren in je Raad-het-getalspel, zodat je het daar kunt gebruiken.

    Dit project is van Hoofdstuk 1 Les 2 Pagina 4: Namen van Spelers Leren.
  1. Open je H1L2-GroetSpelerproject, voer het een paar keer uit en bekijk je code net zo lang totdat je weer weet hoe het werkt. (Op het moment dat je wat namen in je spelerslijst hebt, test dan met een paar namen die wel op de lijst staan en die niet op de lijst staan.)
  2. Fix alle problemen met groet speler. Op het moment dat het correct werkt, exporteer het. (Instructies hieronder.)

    Exporteer blokken

    1. Kies "Blokken exporteren..." van het Snap! menu ().
    2. Selecteer enkel de blokken die je wilt exporteren (in dit geval, enkel groet speler), en klik "OK." Een XML-bestand wordt nu gedownload.
      3. De Snap! Reference Manual vertelt meer over exporteren en importeren.
  3. En dan nu, open je je H2L1-RaadHetGetal project, en importeer je groet speler blok.

    Importeer blokken

    1. Kies "Importeer..." van het Snap! menu ().
    2. Vind het XML-bestand dat je eerder geëxporteerd hebt (waarschijnlijk in de 'download' map) en klik Openen.
    3. Vind de blokken die je wilt. Deze zullen aan het eind van het palet dat deze bevat verschijnen.
    4. Klik op het geïmporteerde blok om ervoor te zorgen dat alles het goed doet. (In dit voorbeeld, zul je waarschijnlijk een error tegenkomen nadat je een naam hebt ingevoerd.)
  4. Kijk naar de code voor groet speler . Kan je zien wat er fout gaat?

Debuggen van afhankelijkheden

Soms, als je een script uit het programma haalt waarin je het hebt gebouwd en het dan ergens anders probeert te gebruiken, gaat er iets fout. Het kan zijn dat het script afhankelijk is van een zelfgemaakt blok dat niet bestaat in het nieuwe programma, of (zoals in dit geval) dat het script afhankelijk is van een globale variabele dat niet gedefinieerd is in het nieuwe programma.

Om dit probleem op te lossen zodat je het geïmporteerde blok goed kunt gebruiken, zul je het globale variabele moeten definiëren in je nieuwe programma. Afhankelijkheden zijn een reden om een script variabele te gebruiken in plaats van een globale variabele indien mogelijk. Dit is zo omdat ze gedefinieerd zijn in hun scripts, en niet verdwijnen wanneer je het script naar een andere plek verplaatst.

  1. Maak de ontbrekende globale variabele, spelerslijst, en zorg dat het programma het initieert als een lege lijst.
    Om een lege lijst aan te maken, klik op de linker pijl op het lijst-blok, zodat er geen nieuwe invoerplekken zijn.

    2. Als spelerslijst wordt geïnitieerd elke keer wanneer het spel opnieuw start, zal het spel nooit een naam onthouden. Je zult dus een ander moment nodig hebben om het spel te starten dan het initialiseren van de spelerslijst. Bijvoorbeeld, je kunt de groene vlag gebruiken om de lijst te initialiseren, en de spatiebalk gebruiken om het Raad-het-getalspel te starten.

  2. Test en Debug. Je programma moet
    • de spelers groeten en hun namen onthouden;
    • het Raad-het-getalspel elke keer te starten met een nieuw geheim getal.
    Fix alle problemen met je code.
    Is er een probleem met de variabele?
    • Check de variabele-initialisatie (daar waar het wordt aangemaakt): Staat de code waar het hoort te staan? Doet de code datgene waarvoor het is bedoeld?
    • Check waar de variabele is gebruikt: Gebruik je de variabele die je bedoelt? Gebruik je het op de manier zoals je verwacht?
