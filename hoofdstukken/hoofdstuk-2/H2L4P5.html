Debuggen is de kunst van het identificeren en oplossen van fouten in computerprogramma's

Tips

  • Niet je hele project verwijderen en opnieuw beginnen. Dit klinkt vanzelfsprekend als je het leest, maar informatica-studenten worden soms boos op hun bugs. Onthoud dat als je 20 regels code hebt en in één van die regels zit een fout, dan is je programma voor 95% correct.
  • Voeg geen code toe zodat je om de bug heen kan werken. Pas de code niet aan tot je helemaal begrijpt waarom het niet doet wat het moet doen, verwijder dan de fout. Denk: "Debug door weg te halen niet door toe te voegen."
  • Wees niet te streng tegen jezelf. Iedereen heeft bugs in zijn programma's; het is een onderdeel van programmeren.
  • Reproduceer het probleem zodat je weet hoe je het opnieuw kan creëren. Dit helpt je om te weten wanneer het probleem is opgelost.
  • Isoleer het stuk code dat zorgt voor de fout. Test stukken code apart zodat je weet wat werkt en wat niet werkt.
  • Identificeer de specifieke oorzaak van de fout in de code. Waarom doet de code het fout?
  • Test altijd je code met verschillende invoeren. Wat als de invoer heel groot, nul, negatief of een kommagetal is? Wat als de gebruiker iets typt wat je niet verwacht? Wat als de gebruiker ergens anders klikt?
  • Houd van je bugs! Ze vormen een interessante puzzel. Soms geeft incorrect gedrag inspiratie voor een ander project:

  • Gebruik om pauzes in je code te plaatsen en op je eigen tempo door de code te gaan.
  • Gebruik blokken in Snap! om informatie over de staat van je programma te krijgen terwijl het bezig is.
  • Gebruik Stapsgewijs Programmaverloop () om te besturen hoe snel Snap! door de blokken van je code gaat.
  • Laat de waardes van variabele en sprite-eigenschappen zien. Je kunt het gedrag zien door op een rapporteur te klikken, het vierkantje naast een variabele () of een sprite-eigenschap () in het palet te vinken. Het kan ook met behulp van .
  • Documenteer je programma met behulp van opmerkingen.
  • Houd je code netjes. Verwijder ongebruikte blokken en gebruik Snap! 's "opruimen" functie om je code te organiseren.
  • Gebruik namen die iets betekenen voor blokken en variabelen. Sommige programmeurs proberen tijd te besparen door hun variabelen namen te geven zoals x. Maar als ze dit vergeten en een andere variabele ook x noemen, kan je bugs krijgen. Dus, om je programma begrijpelijker te maken en makkelijker te debuggen, gebruik je namen die iets betekenen, zoals horizontal position.
  • Gebruik abstractie om complexiteit in je code te verbergen. Abstractie is een manier van denken: je stopt wat details in één deel van je code en zorgt dat die code werkt, dan kan je dat deel vol vertrouwen gebruiken, net alsof het al in Snap! gebouwd zat. Dit soort abstracties laat je stukken code één voor één bouwen en testen, waardoor bugs automatisch geïsoleerd worden.
